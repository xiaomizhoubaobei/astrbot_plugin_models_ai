# 贡献者行为准则

## 我们的承诺

为了营造一个开放和友好的环境，我们作为贡献者和维护者承诺：让参与我们项目和社区的每个人都能享受无骚扰的体验，无论其年龄、体型、残疾、种族、性别特征、性别认同和表达、经验水平、教育程度、社会经济地位、国籍、个人形象、种族、宗教或性取向和身份如何。

## 我们的社区价值观

AstrBot Models AI 插件项目致力于：

* **开放协作**：欢迎所有人参与，无论经验水平如何
* **相互尊重**：尊重每个人的贡献和观点
* **持续学习**：鼓励知识分享和技术交流
* **创新精神**：拥抱新想法和创造性解决方案
* **安全可靠**：确保代码质量和用户安全

## 我们的标准

### 积极行为包括：

* 使用友好和包容的语言
* 尊重不同的观点和经验
* 优雅地接受建设性批评
* 关注对社区最有利的事情
* 对其他社区成员表示同理心
* 帮助新贡献者融入社区
* 提供有意义的反馈和指导
* 尊重他人的时间和努力
* 对技术决策进行建设性讨论

### 不可接受的行为包括：

* 使用性化的语言或图像，以及不受欢迎的性关注或性暗示
* 恶意攻击、侮辱/贬损评论，以及人身或政治攻击
* 公开或私下骚扰
* 未经明确许可发布他人的私人信息，如物理或电子地址
* 在专业环境中被合理认为不适当的其他行为
* 恶意破坏项目或故意引入错误代码
* 拒绝接受他人的贡献而不提供合理的理由
* 歧视性言论或行为
* 威胁、恐吓或暴力行为

## 我们的责任

项目维护者有责任阐明可接受行为的标准，并应对任何不可接受的行为采取适当和公平的纠正措施。

项目维护者有权删除、编辑或拒绝与本行为准则不符的评论、提交、代码、wiki 编辑、问题和其他贡献，并有权暂时或永久禁止任何他们认为行为不当、威胁、冒犯或有害的贡献者。

## 适用范围

本行为准则适用于所有项目空间，包括但不限于：

* GitHub 仓库（Issues、Pull Requests、Discussions、Wikis）
* 项目文档和代码
* 社区聊天室和论坛
* 线上和线下活动
* 任何与项目相关的社交媒体

当个人代表项目或其社区时也适用于公共空间。代表项目或社区的示例包括使用官方项目电子邮件地址、通过官方社交媒体账户发布信息，或在在线或线下活动中担任指定代表。项目维护者可以进一步阐明和明确项目代表。

## 举报机制

### 如何举报

如果您遇到或目睹违反本行为准则的行为，请通过以下方式举报：

* **GitHub Issues**：在仓库中创建一个带有 `conduct` 标签的 Issue
* **邮件**：发送邮件至项目维护者

### 举报内容应包括

* 违规行为的具体描述
* 发生的时间和地点
* 涉及的人员（如果适用）
* 任何相关的截图或链接（如果适用）
* 您的联系信息（可选，但有助于后续沟通）

### 执行流程

1. **接收举报**：项目团队将在 48 小时内确认收到举报
2. **调查**：所有投诉都将被审查和调查
3. **回应**：根据调查结果采取必要和适合情况的回应
4. **保密**：项目团队有义务对事件的报告者保密
5. **跟进**：必要时进行后续跟进，确保问题得到妥善解决

### 可能的执行措施

根据违规行为的严重程度，可能采取以下措施：

* 口头或书面警告
* 要求公开或私下道歉
* 暂时限制参与某些活动
* 暂时或永久禁止参与项目
* 在适当情况下向相关当局报告

## 申诉流程

如果您认为执行决定不公正，可以通过提交 Issue 或发送邮件进行申诉。申诉将在 7 天内得到审查和回应。

## 豁免

本行为准则适用于所有项目贡献者和维护者，没有豁免权。所有社区成员都应遵守相同的标准。
